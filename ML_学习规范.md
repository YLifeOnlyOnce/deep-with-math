# ML 学习与文件组织规范

> 目标：保证内容可复习、可执行、可扩展，避免笔记混乱。

---

## 1. 目录与命名规范

1. 每一章单独一个文件夹，命名格式：`序号_章节名`
   - 示例：`01_数学基础`
2. 每一节单独一个 Markdown 文件，命名格式：`序号_小节名.md`
   - 示例：`01_导数_偏导_梯度.md`
3. 每一节配套一个可视化 Notebook，命名格式：`序号_小节名_可视化.ipynb`
   - 示例：`01_导数_偏导_梯度_可视化.ipynb`
4. 每一节图像统一放在同级 `assets/` 目录。

推荐结构：

```text
01_数学基础/
  README.md
  01_微积分/
    01_导数_偏导_梯度.md
    01_导数_偏导_梯度_可视化.ipynb
    02_链式法则与反向传播直觉.md
    02_链式法则与反向传播直觉_可视化.ipynb
    assets/
```

---

## 2. 每节内容模板（固定）

每节 `.md` 必须包含：

1. 直觉理解
2. 数学定义
3. 推导过程
4. 小例子
5. 图表化理解（含图片）
6. 常见误区
7. 可复述版（考试/面试）
8. 记住一句话（本节最小记忆锚点）

并新增术语要求：

9. 术语与缩写对照（首次出现必须写全称）
   - 格式：`缩写（英文全称，中文名称）`
   - 示例：`PCA（Principal Component Analysis，主成分分析）`
   - 示例：`SVD（Singular Value Decomposition，奇异值分解）`
   - 示例：`MSE（Mean Squared Error，均方误差）`

---

## 3. 公式与可视化规范

1. 公式统一使用 `$...$`（行内）和 `$$...$$`（块级）。
2. 每节至少提供 2~3 张图：
   - 函数形状图 / 等高线图 / 训练轨迹图 / 概率分布图（按主题选）
3. `.md` 中必须嵌入图片路径，保证复习时可直接查看。

4. 文中首次出现专业缩写时，必须给出英文全称与中文名称。

5. 每个关键公式后必须有“文字翻译”（至少 1~3 句），说明：
   - 公式在说什么
   - 每个符号代表什么
   - 在训练/算法中怎么用

6. 讲解采用“公式三明治”结构：
   - 先讲人话（直觉）
   - 再给公式
   - 再做文字拆解（可配一行小例子）

---

## 4. Notebook 规范

1. Notebook 负责“可执行可视化”，并将图片保存到 `assets/`。
2. 图像命名建议：`序号_图主题.png`，例如：`04_chain_rule.png`。
3. 每个 Notebook 首格说明：本节生成哪些图、输出到哪里。

---

## 5. 环境规范（当前项目）

已采用独立虚拟环境：`.venv-jupyter-ml`

核心安装项：
- Python 3.12
- jupyterlab / notebook / ipykernel
- numpy / matplotlib / pandas / seaborn / scipy / scikit-learn

常用命令：

```bash
# 启动 Jupyter Lab
.venv-jupyter-ml/Scripts/python.exe -m jupyter lab

# 执行某个 notebook（批处理）
.venv-jupyter-ml/Scripts/python.exe -m jupyter nbconvert --to notebook --execute --inplace "路径/文件.ipynb"
```

## 6. 知识沉淀规范

1. 每节完成后，把关键点写入 `ML_关键点记录.md`。
2. 将该节关联关系更新到 `ML_知识图谱.md`。
3. 关键点坚持“结论 + 公式 + 误区 + 关联主题”。

- 若你提出新偏好（例如图风格、公式详细程度），优先覆盖本规范。
